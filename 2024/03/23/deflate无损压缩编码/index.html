deflate

DEFLATE 是基于LZ77 算法 + 哈夫曼编码（Huffman Coding）的一种无损数据压缩算法。

deflate blocks

deflate是分块压缩的，raw deflate 仅包含一组块，每一个块(block)都对应一段数据压缩后存储的比特流，块与块之间不要求字节对齐。
每一个块都有一个3bit的头，其格式如下：
bit 0 BFINAL
bit 1-2 BTYPE

如果当前的block是最后一个block时，BFINAL bit会置上。
BTYPE则指明了当前block的类型：
b’00 no compression
b’01 compressed with fixed Huffman codes
b’10 compressed with dynamic Huffman codes
b’11 reserved (error)

no compression block (aka stored block)

no compression block 存放的是没有经过压缩算法处理的原文。

type size description
BFINAL 1 bit b’* (0 or 1)
BTYPE 2 bits b’00
Reserved bits < 8 bits make sure the offset of LEN is byte aligned
LEN 2 bytes number of data bytes in the block
NLEN 2 bytes NLEN = 65535 - LEN, used for the number validation
Literal data LEN in bytes uncompressed data

dynamic Huffman block

fixed Huffman block 是数据经过压缩的块，它是用动态生成的huffman coding tree来进行字符编码的，所以这个动态生成的huffman tree的信息也存储在当前block里。

type size description
BFINAL 1 bit b’* (0 or 1)
BTYPE 2 bits b’10
HLIT 5 bits number of Literal/Length codes - 257 (257 - 286)
HDIST 5 bits number of Distance codes - 1 (1 - 32)
HCLEN 4 bits number of Code Length codes - 4 (4 - 19)
Code Length codes 3 bits * (HCLEN + 4) code lengths for the code length table
literal/length distance table * bits (related to the HLIT, HDIST and the code length table) 2 huffman tree stored
huffman encoded bit stream * bits end with EOB symbol (256)

4 bits HCLEN 表明了存储的Code Length codes的大小，其值范围0-15，对应的存储的code length数量为4-19。
举个例子，如果HCLEN=12，那么前16(12+4)个code length以每个3 bit值存储在block中，表中index 16-18没有存储在block中，这表示index 16-18 对应的code length 14，1，15 没有被用到。而这3 bits的值表示了对code length再进行huffman编码后的code长度，3 bit 值如果为0的话表明当前对应的code length也没有被用到。
假设这16个code length存储的3bit值为：[1,0,5,5,3,0,5,0,4,0,0,4,0,5,0,3]
那么对应的：

这个code length huffman tree是用来解出接下来literal/length distance table里面的两个huffman tree，
分别是literal/length huffman tree和distance huffman tree。

对literal/length的编码：
deflate规范中，长度信息与普通字符共用同一编码空间. 这个编码空间一共有286个编码, 范围是从0至285.
其中0至255 为普通字符编码, 256表示压缩块的结束; 而257至285则表示长度. 长度编码的含义如下表所示:

fixed Huffman block

fixed Huffman block 是数据经过压缩的块，它是用一个定义好的huffman coding tree来进行字符编码的，这个huffman tree并没有存储在当前block里。
